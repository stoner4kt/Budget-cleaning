## ğŸš€ How to Set Up Your Substack Blog Integration

### Step 1: Get Your Substack RSS Feed URL
1. Go to your Substack publication
2. Your RSS feed URL will be: `https://YOUR_USERNAME.substack.com/feed`
3. Replace `YOUR_USERNAME` with your actual Substack username

### Step 2: (Optional) Get RSS2JSON API Key
For better reliability and higher rate limits:
1. Visit https://rss2json.com
2. Sign up for a free account
3. Get your API key from the dashboard
4. Free plan includes: 10,000 requests/day

### Step 3: Update blog.html
Open `blog.html` and find these lines (around line 257-258):

```javascript
const SUBSTACK_RSS_URL = 'https://YOUR_SUBSTACK_URL.substack.com/feed';
const RSS2JSON_API_KEY = 'YOUR_RSS2JSON_API_KEY'; // Optional
```

Replace with your actual values:

```javascript
const SUBSTACK_RSS_URL = 'https://yoursubstack.substack.com/feed';
const RSS2JSON_API_KEY = 'your-actual-api-key-here'; // Or leave as is for no key
```

### Step 4: Update Subscribe Links
Search for `YOUR_SUBSTACK_URL` in blog.html (3 occurrences) and replace with your Substack URL:
- Line 123: Blog error message link
- Line 136: No posts subscribe button
- Line 230: Subscribe CTA button

Replace `https://YOUR_SUBSTACK_URL.substack.com` with your actual Substack URL.

## ğŸ†˜ Troubleshooting

### Blog Posts Not Loading?
1. **Check your Substack URL** - Make sure you replaced `YOUR_SUBSTACK_URL` with your actual username
2. **Verify RSS feed** - Visit `https://yoursubstack.substack.com/feed` in your browser to confirm it works
3. **Check browser console** - Open DevTools (F12) and look for error messages
4. **CORS issues?** - RSS2JSON handles CORS, so this shouldn't be a problem
5. **API rate limit?** - Free tier allows 10,000 requests/day. Get API key if needed.

### Blog Showing "No Posts"?
- Your Substack needs to have at least one published post
- The feed URL must be correct
- RSS2JSON service must be accessible

### Styling Issues?
- Clear browser cache (Ctrl+Shift+Delete)
- Hard refresh page (Ctrl+F5)
- Ensure style.css is loading properly
